RP2040 I2C Slave Configuration Tests
====================================

The bus freeze issue suggests Wire1 slave mode might not be working.
Try these configurations in order:

TEST 1: Disable I2C entirely (verify LEDs work)
------------------------------------------------
In src/main.cpp, line 18:
#define DISABLE_I2C

Upload and verify:
- Serial monitor shows "I2C DISABLED FOR DEBUGGING"
- i2c_scan.py on Raspberry Pi works (no freeze)
- LEDs flash white briefly on startup

If this works, the SCORPIO board and LEDs are fine.


TEST 2: Use Wire (i2c0) with GPIO 0/1
--------------------------------------
In src/main.cpp:
- Line 8: #define I2C_SDA_PIN 0
- Line 9: #define I2C_SCL_PIN 1
- Line 15: // #define USE_WIRE1  (comment it out)
- Line 18: // #define DISABLE_I2C  (comment it out)

Wiring:
- SCORPIO GPIO 0 → Raspberry Pi GPIO 2 (Pin 3, SDA)
- SCORPIO GPIO 1 → Raspberry Pi GPIO 3 (Pin 5, SCL)
- Common GND

Upload and test with i2c_scan.py
If it freezes, slave mode might not be supported.
